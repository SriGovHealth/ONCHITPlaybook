Content Style Guide

Last updated: May 13, 2016

This style guide is a resource for ONC. It supports clear content, consistent formatting, and usability in the Patient Engagement Playbook, leading to a unified ONC voice and a better user experience.

Note: Follow Chicago Manual of Style for anything not covered in this style guide.

Bulleted Lists

  • Capitalize the first word in each bullet
  • Don’t use punctuation (including periods, commas, and semicolons) after bullets, unless 1 bullet includes 2 or more sentences
  • Don’t use “and,” “but,” or “or” before the last bullet
  • Don’t use numbered lists unless the content outlines steps that happen in a specific order

Capitalization

  • Use sentence case for chapter titles and section headers
    Example
    • Yes: Facilitate easy enrollment
    • No: Facilitate Easy Enrollment
  • Use title case for the full playbook title (proper noun), but lowercase for “this playbook” (noun)
    Example
    • Yes: ONC developed the Patient Engagement Playbook
    • No: ONC developed the patient engagement playbook
    Example
    • Yes: ONC developed this playbook
    • No: ONC developed this Playbook
  • Use title case for proper nouns (e.g., Boston Medical Center)
  • Use title case for “Meaningful Use” and always spell it out
    Example
    • Yes: Meaningful Use
    • No: MU, meaningful use

Acronyms and Initialisms

  • Spell out acronyms and initialisms the first time you see them
    Example
    • Yes: Health Information Portability and Accountability Act (HIPAA)
    Example
    • Yes: health information technology (health IT)
  • Don’t include periods in abbreviations or acronyms, except “U.S.”

Numbers

  • Use the symbol “%” instead of spelling out “percent”
  • Use numerals (digits) instead of spelled-out numbers for numbers of any size, except when beginning a sentence or when using “one” to refer to a person. Bullets can begin with numerals.
    Example
    • Yes: 2 categories
    • No: two categories
    Example
    • Yes: 50% black
    • No: Fifty percent black
  • When referring to a range of years, always use full years. Use “to” instead of an en dash.
    Example
    • Yes: 2000 to 2001
    • No: 2000­­–01
  • When referring to age, use “age” or “ages,” not “years old,” “the age of,” “aged,” or just a number. Use “to” instead of an en dash.
    Example
    • Yes: Women ages 18 to 24
    • No: Women ages 18–24
    Example
    • Yes: Adults ages 65 years or older
    • No: Adults aged 65 years or older
    Example
    • Yes: Children ages 7 to 10
    • No: Seven- to ten-year-olds

Punctuation

Periods and commas

  • Use only 1 space between sentences
  • Don’t punctuate page titles
  • Use the Oxford (serial) comma
    Example
    • Yes: poster, iPad, and front desk
    • No: poster, iPad and front desk

Dashes and hyphens

  • Use em dashes (—) instead of double hyphens (--)
  • Use spaces around em dashes
  • In general, only hyphenate compound terms when they function as adjectives before a noun. Exception: always hyphenate compound nouns that start with “self.”
    Example
    • Yes: patient-generated data
    • No: patient generated data
    Example
    • Yes: self-care
    • No: self care
    Examples
    • Yes: The study was high quality, the high-quality study
    • No: The study was high-quality, the high quality study

Miscellaneous

  • Use contractions
    Examples
    • Yes: don’t, isn’t
    • No: do not, is not
  • Use “smart quotes,” not "straight quotes"

Links

Link text and punctuation

  • Write link text as a descriptive phrase or short sentence
    Example
    • Yes: Exchange secure messages with more than 5% of patients
    • No: Click here for information on exchanging secure messages
  • Don’t hyperlink headers (users may not know the header is also a link) — add links only to body text
  • Don’t include punctuation at the end of a link
    Example
    • Yes: Meaningful Use will soon be replaced by Advancing Care Information under the Quality Payment Program. (Here, the period isn’t linked.)
    • No: Meaningful Use will soon be replaced by Advancing Care Information under the Quality Payment Program. (Here, the period is linked.)
  • If the link text includes the title of an initiative or chapter, use title case for the title — but don’t use italics
    Example
    • Yes: See Chapter 2 of this playbook for more information on portal benefits.
    • No: See chapter 2 of this playbook for more information on portal benefits.

Opening links

  • Make links to content within the playbook open in the same tab
  • Make links to content outside the playbook open in a new tab

Exit disclaimer

  • Include the exit disclaimer icon () on links to all non-federal sites, even state government sites
  • Insert 1 space between the end of the hyperlink and the exit disclaimer icon. Add punctuation after the exit disclaimer.
    Example
    • Yes: https://developer.apple.com/healthkit/.

PDF label and size

  • When linking to a PDF, include a file type/size indicator. Use square brackets and an en dash ( – ), with a space in the file size. Punctuation goes outside the brackets. Don’t include final punctuation in the hyperlink.
    Example
    • Yes: http://www.opennotes.org/wp-content/uploads/2012/08/OpenNotes-Fact-Sheet.pdf [PDF – XX KB].
    • No: http://www.opennotes.org/wp-content/uploads/2012/08/OpenNotes-Fact-Sheet.pdf [PDF–XX kB].

References and Attributions

  • Place reference numbers (e.g., footnotes) outside of all punctuation marks except a dash (place them before a dash)
  • Use APA reference style
    Example
    • Yes: Agboola S, Jethwani K, Khateeb K, Moore S, Kvedar J. Heart Failure Remote Monitoring: Evidence From the Retrospective Evaluation of a Real-World Remote Monitoring Program. J Med Internet Res. 2015;17(4):e101. http://www.jmir.org/2015/4/e101.
  • When attributing a quote to a professional, include that person’s name, credential, title, and affiliation
    Example
    • Yes: Joseph C. Kvedar, MD, Vice President, Connected Health, Partners HealthCare
